# omok-online 프로젝트 지침

## 신규 게임 추가 시 필수 체크리스트

앞으로 이 저장소에 게임을 새로 추가할 때는 아래 3가지를 반드시 함께 구현한다. 게임 로직(`src/games/<game>/logic.ts`)과 룸 컴포넌트(`src/components/<Game>Room.tsx`)를 만드는 시점에 같이 처리할 것.

### 1. 게임 시작 전 튜토리얼/규칙 보기 (skip 가능)
- 대국이 실제로 시작되기 전에 규칙 설명을 볼 수 있는 진입점을 제공한다 (모달/오버레이 등).
- 반드시 "건너뛰기(skip)" 선택지를 둬서, 규칙을 아는 사용자는 즉시 게임을 시작할 수 있어야 한다.
- 한 번 skip 했다고 이후 재접속 시에도 강제로 숨길 필요는 없음 — 매 대국 진입 시 선택 가능한 형태를 기본으로 한다.

### 2. 같은 상대와 다른 게임 이어서 즐기기
다음 두 가지 방식을 모두 지원한다.
- **메인 페이지에서 세션 생성**: 메인 페이지에서 "상대와 세션 만들기" 같은 기능으로, 같은 상대와 여러 게임을 오갈 수 있는 그룹(세션) 단위를 만들 수 있게 한다.
- **재대결 신청 시 다른 게임 선택지 추가**: 기존 재대결 기능(`game_rooms.rematch` jsonb 컬럼, 각 `*Room.tsx`의 rematch 로직 참고)에 "같은 게임으로 재대결" 외에 "다른 게임으로 재대결"을 선택할 수 있는 옵션을 추가한다.

### 3. "같은 화면에서 둘이 두기" (로컬 2인 플레이)
- 모든 게임은 온라인 대전과 별개로, 한 화면/한 기기에서 두 사람이 번갈아 두는 로컬 모드를 지원해야 한다.
- 오목의 `src/app/room/local/page.tsx`를 참고 패턴으로 삼는다 (Supabase 연결 없이 로컬 상태만으로 진행, 종료 시 `saveLocalRecord`로 기록 저장).
- 새 게임 로직이 이미 순수 함수(`applyMove` 등)로 분리되어 있다면 그대로 재사용해서 로컬 룸 페이지를 만들면 된다.
